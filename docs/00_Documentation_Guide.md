# 📂 문서 가이드 (Documentation Guide)

이 폴더(`docs/`)는 프로젝트의 모든 기획, 명세, 개발 로그를 관리하는 핵심 저장소입니다.
AI Agent와 개발자는 **항상 이 문서를 기준**으로 소통하고 개발을 진행해야 합니다.

---

## 📄 파일별 역할 및 규칙

### **`01_Project_Spec_and_Auth.md` (프로젝트 명세 및 인증)**
*   **역할**: 프로젝트의 기술 스택(Tech Stack), DB 접속 정보(Credentials), 환경 변수 등을 관리하는 보안/설정 문서입니다.
*   **수정 규칙**: 기술 스택 변경 시 업데이트합니다. **비밀번호 등 보안 정보는 외부에 유출되지 않도록 주의하세요.**

### **`02_Original_Game_Design.md` (초기 기획서 원본)**
*   **역할**: 게임의 오리지널 기획 의도와 초기 설계를 담고 있는 **불변의 기준점(Base Truth)**입니다.
*   **⚠️ 중요 규칙**: **사용자가 명시적으로 수정을 지시하지 않는 한, 이 파일의 내용은 절대 변경하지 않습니다.**
*   **활용**: `03`번 파일(현재 구현 상태)과 비교하여, "기획되었으나 아직 구현되지 않은 기능"을 파악하는 용도로 사용합니다.

### **`03_Live_Game_Mechanics.md` (메커니즘 인덱스)**
*   **역할**: `docs/mechanics/` 폴더 내의 상세 문서들로 연결되는 **목차(Index)** 파일입니다.
*   **구조**: 게임 규칙이 방대해짐에 따라 **스탯, 행동, 친구 시스템** 등을 별도 파일로 분리하였습니다.
*   **활용**: 특정 규칙(예: 친구 확률)을 찾을 때 이 파일의 링크를 타고 들어가세요.

### **`docs/mechanics/` (상세 규칙 폴더)**
*   **`01_Stats_and_Formulas.md`**: 4대 스탯, 시험 점수, 노화 시스템.
*   **`02_Action_System.md`**: 행동 비용, 스트레스/체력 공식.
*   **`03_Friend_System.md`**: 친구 등급, 확률, 상호작용.

### **`04_Dev_Log_and_Plan.md` (개발 로그 및 계획)**
*   **역할**: 일자별 상세 개발 내역(Bugfix, Feature)과 향후 개발 계획(To-Do)을 기록합니다.
*   **수정 규칙**: 작업이 끝날 때마다 오늘 한 일을 상세히 기록하고, 남은 작업 목록을 갱신합니다.

---

## 🔄 개발 프로세스 가이드
1.  **기획 비교**: `02`(기획) vs `03`(구현)을 비교하여 미구현 기능을 식별합니다.
2.  **작업 진행**: 식별된 기능이나 버그 수정을 진행합니다.
3.  **동기화**: 작업 완료 후 변경된 수치는 `03`에, 작업 내역은 `04`에 기록합니다.
